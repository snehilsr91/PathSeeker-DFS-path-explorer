Detailed Explanation of DFS for Path Finding

Depth-First Search (DFS) is a fundamental graph traversal algorithm...

Algorithm Steps for Finding All Paths

  1. Initialize an empty list to store all found paths.
  2. Create a `visited` set/array to keep track of nodes visited *in the current path* to avoid cycles within that specific path attempt.
  3. Define a recursive DFS helper function: `dfs_util(current_node, destination_node, current_path, visited_in_path)`
    • Mark `current_node` as visited (add to `visited_in_path`).
    • Add `current_node` to `current_path`.
    • If `current_node` is the `destination_node`:
      • A path is found! Add a copy of `current_path` to the list of all paths.
      • (Important for finding multiple paths: Do NOT return immediately. Backtrack to find other alternatives).
    • Else (not destination):
      • For each neighbor `v` of `current_node`:
        • If `v` is NOT in `visited_in_path`:
          • Recursively call `dfs_util(v, destination_node, current_path, visited_in_path)`.
    • Backtrack:
      • Remove `current_node` from `current_path`.
      • Unmark `current_node` (remove from `visited_in_path`). This is CRUCIAL to allow this node to be part of other paths.
  4. Start the process by calling `dfs_util(source_node, destination_node, initial_empty_path, initial_empty_visited_set)`.

Visualization Aspects

When visualizing: - Highlight nodes as they are visited. - Highlight edges as they are traversed. - Show a different highlight for nodes during backtracking. - Clearly indicate when a complete path is found.
